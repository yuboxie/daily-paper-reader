1 Introduction 

Conversational agents are increasingly explored as scalable mental health support tools, offering on-demand access, anonymity, and reduced cost compared to traditional care [19, 22, 23, 44]. Advances in large language models (LLMs), such as ChatGPT, now enable open-ended, context-sensitive conversations, expanding the expressive capacity of mental health chatbots [ 27 , 30 ]. At the same time, structured therapeutic frameworks can be embedded into conversational systems. The Self-Attachment Technique (SAT) [ 9 ] is a self-administered, attachment-based approach that supports emotion regulation and self-compassion by cultivating an imaginative affectional bond between the adult self and the childhood self of an individual [ 4, 12 ]. Prior SAT chatbots using rule-based or classification-driven dialogue demonstrated promising empathy and engagement, but were limited in flexibility and scalability [4, 12]. Despite these advances, most mental health chatbots emphasize feasibility and acceptability while relying on shallow interaction patterns and short-term evaluations, leaving open questions about how system architecture shapes perceived therapeutic quality [ 19 , 23 , 44 ]. Research highlights the importance of personalization, empathic responses, and sustained engagement [ 19 , 22 ], yet existing work rarely compares architectural paradigms under controlled conditions [ 47 ]. Recent proposals such as multi-agent orchestration and finite state machines offer ways to structure complex LLM behavior [ 47 ], but their impact on therapeutic naturalness, authenticity, and engagement, particularly for structured interventions like SAT, remains underexplored. To address these gaps, we conducted a between-subjects randomized controlled trial (RCT) examining how LLM-based architectural design influences perceived therapeutic quality in a SAT-informed chatbot. Using GPT-4o [ 31 ] as the base model across all conditions, we compared three architectures. The Intervention employed a multi-agent finite state machine aligned with SAT stages, incorporating explicit state transitions, a SAT knowledge base, and shared long-term memory among the agents. The Active Control used a single agent with equivalent prompts and SAT access. The Placebo Control used GPT-4o without SAT knowledge or structure, resembling a generic conversational companion. System architecture and prompts were authored in English, while deployment targeted a Farsi-speaking user population. We deployed the system for eight days with 𝑁 = 66 participants (Intervention: 22, Control: 23, Placebo: 21), who completed demographic measures, interacted with their assigned chatbot, and completed post-study surveys. The intervention was perceived as significantly more natural and human-like than both control conditions (F=7.017, p=0.0018, 

𝜂 2=0.187), with mostly higher but non-significant scores on other therapeutic metrics, such as trust, memory coherence, user satisfaction, fewer off-topic responses, and ease of use. Chat logs showed the intervention produced more but shorter messages (459 messages, ∼230 characters) than the active control (336 messages, ∼409 characters) and placebo control (206 messages, ∼635 characters), indicating a more interactive dialogue style. Findings demonstrate that architectural choices, beyond prompt design alone, meaningfully shape user experience in LLM-based therapeutic chatbots. 2 Related Work 

The Self-Attachment Technique [ 9, 10 ], grounded in Bowlby’s attachment theory [ 5, 6], helps users regulate emotional distress by alternating between care-seeking and care-giving roles. Early SAT chatbots operationalize this with rule-based dialogue and emotion classification to provide empathetic responses [ 4, 11 , 26 ], enforcing structure but remaining limited in generative flexibility, contextual adaptation, long-term personalization, and multilingual support. Similarly, other therapeutic chatbots, including CBT systems [ 19 , 22 ], Evebot [ 46 ], GPT-2–based therapy bots [ 45 ], and student distress-detection frameworks [ 32 ], combine emotion recognition with neural generation but generally lack explicit therapeutic state modeling, memory, and robustness. The emergence of large language models has enabled richer human–AI collaboration and motivated hybrid neuro-symbolic architectures. Human-in-the-loop systems support cognitive restructuring [ 38 ], yet CAPE-II evaluations report failures in crisis handling and boundary maintenance [ 39 ], and excessive trust can amplify harm [ 7, 40 ]. FSM-based systems like MindfulDiary [ 24 ] and ChaCha [ 37 ] enforce structured progression while allowing free expression, multi-agent frameworks improve robustness and empathy [21 , 47 ], EmoAgent monitors user state to reduce deterioration [ 33 ], and LLMs serve as scalable front-ends for structured intake and assessment [ 36 ]. Compared to early SAT bots [ 4 , 12 ], these architectures provide stronger guarantees of consistency, safety, and flexibility. Emotion modeling and personalization are central to therapeutic effectiveness, optimizing empathy, fluency, and novelty [ 3], adapting interventions via real-time mood detection (EMMA; [ 17 ]), and strengthening therapeutic bonds [25 ]. RCTs validate their impact: Woebot [ 14 ] reduced symptoms compared to self-help, while modality and interaction style influence loneliness and emotional dependence [ 13 ]; other studies highlight emotional resilience [ 28 ], humor [ 42 ], and just-in-time interventions [ 17 ]. Farsi-language systems remain scarce: early Persian SAT chatbots report positive engagement [ 12 ], while HamRaz and HamRazEval support culturally grounded therapy [ 2 ] and PersianLLaMA provides a foundation for generative modeling [ 1], though personalization and fluency remain limited. Our work synthesizes these strands by embedding a context-aware LLM within a multi-agent, FSM-driven SAT architecture, advancing prior SAT and Persian systems with structured progression, safety-aware generation, long-term engagement, and one of the first RCT-based evaluations of a Farsi SAT chatbot. 

3 Method 

System Design and Architecture. To investigate how conversational structure and adaptivity influence therapeutic engagement, we developed a three-tier conversational AI system that delivers the Self-Attachment Technique within a Randomized Controlled Trial framework. The study included three experimental conditions: (1) a finite state machine (FSM) driven multi-agent system with long-term memory intervention (Alpha), (2) a single-agent system with a knowledge-base as an active control (Beta), and (3) a minimal single-agent placebo (Gamma). All versions run on a Django REST backend in Python and employ OpenAI’s GPT-4o [31 ] as the conversational engine. User interactions occurred in Farsi, supported by a specialized response retrieval system for intent classification, while prompts and conversational structures were authored in English. To maximize ecological validity, participants accessed the intervention via a React.js web interface, identical across all three conditions. 

Experimental Conditions and Randomization. Upon registration, participants were randomly assigned to one of three groups and blinded to their condition to minimize expectancy effects. All systems employed the same LLM and shared interface, response style, and interaction modality. (1) Alpha implements a 12-state therapeutic protocol based on the SAT stages [ 12 ], while supporting open-ended conversation throughout the flow. At each stage, dedicated 4 Elahimanesh et al. agents guide state transitions using LLM-driven decision logic, while maintaining open-ended dialogue to engage users and smoothly navigate the conversation to the next stage. To ensure consistency among the agents, the system utilizes a shared long-term conversational memory, generated via prompt-guided LLM summarization to preserve emotional states, personal context, and salient experiences. Additionally, it incorporates an agent using an adaptive Retrieval-Augmented Generation (RAG) mechanism to deliver personalized therapeutic exercises; (2) Beta delivers the identical SAT content and daily exercises but without FSM enforcement, relying solely on a single comprehensive system prompt to implicitly guide the conversation through similar phases within the LLM’s context window. To isolate the effect of architecture, this version uses the exact same prompts as Alpha, collapsed into a single sequential prompt; 

(3) Gamma provides a minimal conversational single-agent, controlling for the “digital companion” effect. It uses a stripped-down prompt with no SAT knowledge, no exercise recommendation engine, and no structured therapeutic goals, representing a near–pure instantiation of the LLM. 

Conversational Structure: The Finite State Machine. A key contribution of the Alpha system is its explicit modeling of the therapeutic process. The FSM divides the interaction into four logical Superstates or phases: Initiation 

(Greeting/Emotion Assessment), Exploration (Event Processing), Intervention (Exercise Delivery), and Conclusion .Unlike rigid rule-based chatbots that rely on keyword matching, the Alpha system employs two advanced transition mechanisms to maintain naturalness: (1) LLM-as-Judge Sufficiency Detection [ 43 , 48 ], where a dedicated prompt evaluates whether accumulated user responses satisfy emotional and informational criteria for the current state, ensuring meaningful expression of the feelings and events before state change; (2) Intent-Based Routing , which classifies over 40 affirmative/negative Farsi linguistic patterns to route the user to the correct next state. 

Knowledge Injection and Adaptive RAG. A comprehensive SAT knowledge base, containing theoretical foundations and 27 exercises, was embedded into the system prompts for both Alpha and Beta. Delivery, however, differed: Beta used a static schedule, selecting exercises from a fixed pool based on the agent, while Alpha employed a hierarchical 

Adaptive RAG mechanism. This process consists of two phases: (1) Constraint-Based Filtering: The system first retrieves a valid candidate set by filtering the exercise database according to the user’s protocol timeline (Intervention Day 1-8) and therapeutic stage (Beginning, Intermediate, or Advanced). (2) Semantic Selection: A ’Selector’ LLM then acts as a re-ranker, analyzing the user’s current emotional state and summarized memory to choose the single most contextually appropriate exercise from the candidate set. It then generates a personalized version of that exercise tailored to the user’s context, minimizing repetition and maximizing personalization. 

Longitudinal Memory and Progress Tracking. Alpha maintains structured long-term memory, creating concise summaries every three messages via prompt-guided LLM abstraction to preserve emotional patterns, preferences, and biographical context. A calendar-based model decouples timeline from interaction frequency, ensuring participants resume the correct stage (capped at Day 8) and follow the protocol consistently from beginner to advanced levels. 

4 Experiment and Results

Ethical approval for the user study was obtained from the institution of one of the authors. Participants were recruited by word of mouth and also from Farsi-speaking communities via social media (Twitter, Telegram), participated voluntarily, provided informed consent, and were informed that the system was not a replacement for professional mental health care; logs and survey responses were anonymized. Interaction Dynamics: Why Architecture Matters. To understand why the multi-agent system achieved higher perceived naturalness, we analyzed chat logs. One key factor appears to be turn-taking, pacing, and message length. Normalized by group size, Alpha participants generated the most agent messages per person ( ≈ 20 .9) versus Beta (≈ 14 .6) and Gamma ( ≈ 9.8), a 43% and 113% increase. Alpha’s agent messages were shorter (229.8 characters) than Beta (408.7) and Gamma (635.1), shifting from long, lecture-style responses to rapid, dialogic exchanges that closely emulate human therapeutic conversation. User behavior mirrored this pattern: Alpha participants sent shorter messages on average (29.0 characters) than Beta (38.9) and Gamma (42.8), and the agent-to-user message length ratio was lowest in Alpha (7.9:1) versus Beta (10.5:1) and Gamma (13.4:1), indicating a more balanced, interactive dynamic. 6 Elahimanesh et al. In addition, importantly the FSM architecture in Alpha, explicitly structured around therapeutic stages, naturally reflects the reasoning of a human therapist, guiding the conversation through problem identification, exploration, and resolution. Multi-agent coordination keeps each agent focused on its stage, while long-term memory preserves context within and across sessions, enhancing continuity and relevance. A structured therapeutic knowledge base ensures contextually appropriate guidance. Beta’s prompting and knowledge contributed to its intermediate performance, and Gamma’s minimal design performed lowest. Together, these elements suggest that Alpha’s design, by mimicking human conversational dynamics, multi-agent specialization, memory retention, and grounded expertise, directly drives higher perceived naturalness and overall interaction quality. 

5 Discussion 

Our findings show that architectural choices in LLM-based therapeutic chatbots strongly shape user experience, beyond prompt engineering and toward fundamental questions of system structure, due to the staged, process-driven nature of therapy. Multi-agent architectures with explicit state management produced more natural interactions than single-agent or unguided LLMs, indicating that effective chatbots require careful orchestration alongside content delivery. 

Conversational Naturalness. The intervention’s advantage in perceived naturalness reflects more than stylistic preference. Natural dialogue aligns with person-centered care, where interaction quality predicts outcomes [ 29 ]. Chat logs show that the multi-agent system produced shorter, more interactive exchanges, shifting from monologic responses to genuine dialogue and challenging assumptions that chatbots should maximize information per turn. Instead, effective conversational AI should mirror human dialogue rhythms through frequent turn-taking and collaborative meaning-making. By distributing therapeutic stages across specialized agents, the architecture likely enabled this dialogic pattern, with long-term memory further contributing by preserving users’ feelings and experiences across sessions. Notably, gains in naturalness did not extend to empathy, trust, or personalization, suggesting that these are related yet distinct dimensions of the therapeutic alliance. Architectural improvements that enhance naturalness alone may therefore be insufficient to foster emotional attunement, which may depend more strongly on prompt design or response content. 

Structural Scaffolding and Design Implications. Across most interaction metrics, including naturalness, trust, memory coherence, user satisfaction, off-topic responses, and ease of use, performance consistently followed Alpha 

> Beta > Gamma. This pattern highlights the role of therapeutic structure and architectural scaffolding in shaping perceived interaction quality, even without user awareness of system design. Stronger structural guidance supports coherent dialogue, relevant questioning, and alignment with therapeutic goals, without limiting conversational flexibility. 

Limitations and Future Directions. Our study highlights several limitations and directions for future work: (1) 

the eight-day deployment captured only early user impressions, leaving long-term engagement, outcomes, and the development of user-agent relationships unresolved, including whether perceived naturalness translates into sustained use or longer-term benefits; (2) the use of single-item measures for empathy, trust, and personalization limits construct validity, and future studies should employ validated multi-item instruments (e.g., the Working Alliance Inventory [18 ]) and examine how architectural effects vary across user characteristics; (3) the focus on Persian-speaking, highly educated participants (93.9% holding at least a Bachelor’s degree) and an English-authored system architecture constrains cross-linguistic and broader population generalizability; (4) therapeutic outcomes and skill acquisition were not directly assessed, underscoring the need to link subjective interaction quality to objective behavioral and clinical measures; and (5) the collected chatbot interaction data represent a valuable therapeutic dataset that, when combined with other sources, could support LLM alignment using methods such as PPO or DPO [34, 49]. 6 Conclusion 

Architectural design strongly shaped user experience in our therapeutic LLM chatbot. Multi-agent, memory-augmented systems with explicit state management produced more natural, human-like interactions than single-agent or unguided alternatives, with higher (though non-significant) relational ratings. By holding content constant, we isolated structural effects often conflated with content. These results suggest that advancing therapeutic AI requires not only capable LLMs and refined prompts, but also careful architectural orchestration to support structured therapeutic progression.

Appendices A Finite State Machine Specification 

The Alpha (Intervention) system is operated using a programmatic Finite State Machine, comprising 12 distinct states, based on the flowchart of the SAT [ 12 ] algorithm. This architecture enforces the therapeutic structure of the Self-Attachment Technique, while maintaining conversational flexibility through LLM decision nodes. The FSM is divided into four logical phases: Initiation, Exploration, Intervention, and Conclusion. 

A.1 Transition Mechanisms 

State transitions are governed by two complementary mechanisms designed to balance protocol adherence with flexible, naturalistic interaction. (1) LLM-as-Judge Sufficiency Detection. For conversational states that require qualitative and experiential information (such as Emotion Assessment , Event Exploration ), we adopt an LLM-as-judge paradigm. In this setup, a dedicated judging prompt instructs the LLM to evaluate whether the accumulated conversation history sufficiently satisfies the informational and emotional criteria defined for the current state. Transitions occur only when the LLM-as-judge determines that the user has meaningfully articulated their emotional state or relevant experiences, enabling adaptive progression without reliance on fixed turn counts. (2) Intent Classification via Fine-Tuned BERT. For decision points requiring binary or categorical user input (such as accepting or declining an exercise), we employ an intent classification module based on BERT [ 8]. The model was fine-tuned on Farsi-language interaction data to recognize user intent, including over 40 affirmative and 13 negative linguistic patterns (such as “bale” , “are” , “begoo” ). This approach enables robust intent detection while avoiding rigid keyword matching, thereby supporting fluent and user-driven conversational flow. 

A.2 Detailed State Descriptions 

A.2.1 Phase I: Initiation and Assessment. 

• State 1: GREETING_FORMALITY_NAME Objective: Establish rapport and configure user preferences. Logic: 

The agent greets the user and optionally requests their name and formality preference (formal vs. informal). The transition occurs once the Judge confirms these parameters are set or explicitly declined by the user. 

• State 2: EMOTION Objective: Identify the user’s current emotional state. Logic: The agent employs active listening to validate feelings. A strictly enforced constraint requires a minimum of two user messages in this state to prevent premature transitions, ensuring the user feels heard before moving forward. 

• State 3: EMOTION_DECIDER (Internal Node) Objective: Route the conversation based on sentiment analysis. 

Logic: An LLM classifier analyzes the dialogue history to categorize the user’s state as Positive or Negative .

– Negative: Transitions to SUPER_STATE_EVENT for processing triggers. 

– Positive: Transitions directly to ASK_EXERCISE , skipping the event exploration phase. 

A.2.2 Phase II: Exploration (Trigger Processing). 

• State 4: SUPER_STATE_EVENT Objective: Explore the antecedents of negative emotions. Logic: Accessible only via the Negative path. The agent probes for specific events or triggers associated with the user’s distress. Similar to the Emotion state, a minimum two-message constraint is enforced to ensure adequate exploration depth. • State 5: OPEN_ENDED_CONVERSATION Objective: Provide a safe space for deep elaboration. Logic: If the user indicates a need to vent or elaborate further during State 4, the system transitions here. This state allows for free-form dialogue (minimum 4 messages) before gently steering the user back to the intervention track. 

A.2.3 Phase III: Therapeutic Intervention. 

• State 6: ASK_EXERCISE Objective: Obtain informed consent for the intervention. Logic: The agent proposes a therapeutic exercise. The Response Retriever classifies the user’s reply; affirmative responses lead to suggestion, while negative responses route to the closing phase. 

• State 7: EXERCISE_SUGGESTION Objective: Deliver a context-aware SAT exercise. Logic: This state utilizes the RAG system to select an exercise. The selection algorithm filters by the user’s specific intervention day (such as Day 1: Exercises 1-3) and therapeutic stage (Beginning/Intermediate/Advanced), then uses the LLM to choose the specific activity that best aligns with the user’s summarized emotional context. 

• State 8: EXERCISE_EXPLANATION Objective: Guide the user through the selected practice. Logic: The system provides step-by-step instructions. The user can request clarification, proceed to feedback, or request a different exercise (looping back to State 10). 

• State 9: FEEDBACK Objective: Debrief and consolidate learning. Logic: The agent asks the user to reflect on their experience immediately post-exercise. 

• State 10: LIKE_ANOTHER_EXERCISE Objective: Iterative intervention loop. Logic: The system offers additional exercises. An affirmative response loops the FSM back to EXERCISE_SUGGESTION , allowing for multi-exercise sessions; a negative response transitions to termination. 

A.2.4 Phase IV: Conclusion. 

• State 11: THANKS Objective: Session closure and positive reinforcement. Logic: The agent provides concluding remarks and encouragement.

## 摘要
现代语言模型在有限的上下文内进行推理，这一固有约束构成了长程推理的根本障碍。我们将递归视为克服这一障碍的核心原则，并提出了递归模型作为一种极简实现，其中模型可以在隔离的上下文中递归地调用自身来解决子任务。我们证明了任何可计算问题都允许递归分解，其中每个子任务所需的活动上下文仅为标准自回归模型的指数级缩小；这严格优于任何局限于单一序列的上下文管理方法（如摘要）。我们进一步将该框架推广到具有任意上下文处理和控制流的现代智能体系统，并证明递归模型可以在这一更广泛的类别中实现最优能力。在实验上，我们训练了一个 3B 模型进行递归推理，并在布尔可满足性（Boolean satisfiability）任务上进行了评估，这是一项需要长程组合搜索的任务，实验结果表明该模型显著优于前沿大语言模型（LLMs）。

## Abstract
Modern language models reason within bounded context, an inherent constraint that poses a fundamental barrier to long-horizon reasoning. We identify recursion as a core principle for overcoming this barrier, and propose recursive models as a minimal realization, where the model can recursively invoke itself to solve subtasks in isolated contexts. We prove that any computable problem admits a recursive decomposition in which each subtask requires only exponentially smaller active context than standard autoregressive models; this strictly surpasses any context management approach confined to a single sequence, such as summarization. We further generalize our framework to modern agentic systems with arbitrary context processing and control flows, and prove that recursive models can achieve optimal power within this broader class. Experimentally, we train a 3B model to reason recursively and evaluate on Boolean satisfiability, a task requiring long-horizon combinatorial search, where it significantly outperforms frontier LLMs.

### 2. 论文提出的方法论
*   **核心思想**：构建**递归模型（Recursive Model, RCM）**。它由一个基础生成器（LLM）和两个极简工具组成：`call`（调用）和 `return`（返回）。
*   **关键技术细节**：
    *   **上下文栈（Context Stack）**：模型维护一个序列栈。当前生成的序列是“活动上下文”，而挂起的父级上下文被存储在外部。
    *   **Call 机制**：当模型生成 `<call>子任务</call>` 时，系统创建一个全新的、隔离的上下文来解决该子任务。
    *   **Return 机制**：子任务完成后，模型生成 `<return>答案</return>`，系统丢弃子任务的中间推理过程，仅将最终答案传回父级上下文。
    *   **局部与全局空间分离**：局部空间（活动上下文）保持恒定且较小，而全局空间（递归深度）可以无限扩展。
*   **理论证明**：论文证明了任何可计算问题都存在递归分解。递归模型在 $O(S(n))$ 的局部空间下，可以解决需要指数级时间 $2^{O(S(n))}$ 的问题，这在计算能力上指数级优于传统的单序列模型（如摘要法）。

*   **性能飞跃**：仅有 3B 参数的递归模型在 SAT 任务上的表现显著优于 235B 的 Qwen3 和 GPT-4o。在简单和中等任务上准确率分别达到 98% 和 95%。

### 7. 优点
*   **理论严密**：结合了计算复杂度理论（如 ATM 理论），为 LLM 的上下文管理提供了坚实的数学基础。

*   **推理高效**：由于活动上下文短，每生成一个 token 的计算量（FLOPs）远低于处理超长上下文的模型。

### 8. 不足与局限
*   **任务单一性**：目前仅在 SAT 任务上进行了验证，尚未在编程、数学证明或通用长文本规划等更多样化的任务上测试。

*   **训练数据依赖**：需要高质量的递归分解轨迹进行微调，这种数据的获取成本可能较高。

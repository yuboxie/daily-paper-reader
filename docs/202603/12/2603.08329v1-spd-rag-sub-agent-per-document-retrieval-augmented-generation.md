## 摘要
回答复杂的现实世界查询通常需要综合散布在海量文档语料库中的事实。在这些场景下，标准的检索增强生成 (RAG) 流水线面临证据覆盖不全的问题，而长上下文大语言模型 (LLM) 在处理海量输入时难以进行可靠的推理。我们提出了 SPD-RAG，这是一种用于详尽跨文档问答的分层多代理框架，它沿文档轴对问题进行分解。每个文档由一个专门的文档级代理处理，该代理仅针对其自身内容运行，从而实现聚焦检索；同时，协调器将任务分派给相关代理并汇总其部分答案。代理的输出通过一个受 Token 限制的合成层合并部分答案来进行综合（该层支持针对海量语料库的递归 Map-Reduce）。这种具有集中融合功能的文档级专业化提高了异构多文档设置下的可扩展性和回答质量，同时构建了一个模块化、可扩展的检索流水线。在用于长上下文多文档问答的 LOONG 基准测试（EMNLP 2024）中，SPD-RAG 获得了 58.1 的平均分（GPT-5 评估），优于普通 RAG (33.0) 和代理式 RAG (32.8)，而其 API 成本仅为全上下文基线 (68.0) 的 38%。

## Abstract
Answering complex, real-world queries often requires synthesizing facts scattered across vast document corpora. In these settings, standard retrieval-augmented generation (RAG) pipelines suffer from incomplete evidence coverage, while long-context large language models (LLMs) struggle to reason reliably over massive inputs. We introduce SPD-RAG, a hierarchical multi-agent framework for exhaustive cross-document question answering that decomposes the problem along the document axis. Each document is processed by a dedicated document-level agent operating only on its own content, enabling focused retrieval, while a coordinator dispatches tasks to relevant agents and aggregates their partial answers. Agent outputs are synthesized by merging partial answers through a token-bounded synthesis layer (which supports recursive map-reduce for massive corpora). This document-level specialization with centralized fusion improves scalability and answer quality in heterogeneous multidocument settings while yielding a modular, extensible retrieval pipeline. On the LOONG benchmark (EMNLP 2024) for long-context multi-document QA, SPD-RAG achieves an Avg Score of 58.1 (GPT-5 evaluation), outperforming Normal RAG (33.0) and Agentic RAG (32.8) while using only 38% of the API cost of a full-context baseline (68.0).

### 1. 核心问题与研究动机
*   **核心问题**：在处理需要综合多个长文档信息的复杂查询时，传统 RAG 存在“证据覆盖不全”的问题（受限于 Top-K 检索），而长上下文 LLM 则存在“推理质量随长度增加而下降”（Lost in the Middle）的问题。
*   **研究动机**：现实场景（如财务审计、学术综述）往往要求“不遗漏任何文档”的详尽分析。现有的系统难以在保持高召回率的同时，兼顾推理的准确性和成本效益。

### 2. 方法论：核心思想与技术细节
SPD-RAG 的核心思想是**沿“文档轴”而非“任务轴”分解问题**，将语料库中的每个文档分配给一个专门的子智能体。其架构分为三层：
*   **协调层 (Coordination Layer)**：由一个高级智能体（如 Gemini 2.5 Pro）将用户查询分解为“共享指令集”（具体的提取任务）和“合成指令”（指导最后如何汇总）。
*   **并行检索层 (Parallel Retrieval Layer)**：为每个文档部署一个轻量级子智能体（如 Gemini 2.5 Flash）。每个子智能体在自己的文档“孤岛”内进行独立的 RAG 循环（搜索+推理），确保局部信息提取的详尽性，且互不干扰。
*   **合成层 (Synthesis Layer)**：采用**递归 Map-Reduce** 机制。
    *   使用聚类算法（凝聚层次聚类）根据语义相似度对各子智能体的发现进行分组。
    *   在 Token 预算限制内，递归地合并相似信息，直到生成最终答案。这种设计使其能够处理理论上无限规模的语料库。

### 3. 实验设计
*   **数据集/场景**：使用 **LOONG 基准测试**（EMNLP 2024），包含金融报告和学术论文两大领域，平均每个案例涉及 11 个文档，上下文长度从 10K 到 250K+ Token。
*   **任务类型**：涵盖亮点定位 (Spotlight Locating)、对比 (Comparison)、聚类 (Clustering) 和推理链 (Chain of Reasoning)。
*   **对比方法**：
    1.  **Baseline (Full Context)**：将所有文档拼接后直接输入长文本模型（作为性能上限）。
    2.  **Normal RAG**：标准的 Top-K 向量检索。
    3.  **Agentic RAG**：在全局索引上进行迭代检索的单智能体系统。

### 6. 主要结论与发现
*   **性能卓越**：SPD-RAG 的平均得分为 **58.1**，远超普通 RAG (33.0) 和 Agentic RAG (32.8)。
*   **成本效益高**：SPD-RAG 达到了全上下文基线性能的 **85.4%**，但 API 成本仅为其 **37.9%**。
*   **擅长复杂任务**：在需要跨文档综合信息的“聚类”和“推理链”任务中，SPD-RAG 的提升最为显著（比普通 RAG 高出 40.5 分）。
*   **解决学术论文难题**：传统 RAG 在学术论文场景下几乎完全失效（PR% 为 0），而 SPD-RAG 成功实现了性能修复。

### 7. 优点与亮点
*   **文档级专业化**：通过将检索限制在单文档内，消除了跨文档的干扰块，提高了局部提取的精度。
*   **高并行性**：所有文档子智能体并行运行，理论上可以随文档数量线性扩展。
*   **递归合成**：相似度引导的递归合并机制有效解决了长文本模型在处理大量中间报告时的“信息丢失”问题。
*   **模块化与可扩展性**：可以根据预算灵活更换不同层级的模型。

### 8. 不足与局限
*   **延迟开销**：由于需要经过协调、检索、合成三个串行阶段，其端到端延迟（54.8s）略高于普通 RAG。
*   **依赖协调器**：如果协调层生成的子任务描述不够准确，会导致底层子智能体提取信息不全。
*   **递归路径未充分验证**：在 LOONG 测试中，由于 Gemini 的上下文窗口足够大，递归合成路径并未被真正激活（一次性合成了），其在数千个文档规模下的稳定性仍需实证。
*   **领域局限**：目前仅在金融和学术领域验证，法律、医疗等更具结构特殊性的领域表现尚待观察。

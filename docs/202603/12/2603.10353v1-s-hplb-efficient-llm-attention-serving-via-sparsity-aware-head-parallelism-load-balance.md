## 摘要
随着大语言模型（LLM）规模的增加和上下文长度的扩展，注意力计算已成为 LLM 推理服务中的关键性能瓶颈。为了实现快速的注意力计算，近期的实践通常在多个 GPU 上对注意力头进行并行化处理，并广泛采用注意力稀疏化技术来减少计算量——即在预设的稀疏度预算下选择性地计算注意力对的子集。在本文中，我们注意到 LLM 模型的注意力头通常表现出异构且稳定的稀疏弹性，这促使我们实施头自适应的稀疏度预算，以在保持高推理质量的同时获得更高的效率。然而，从系统层面来看，由于稀疏程度不同，不同头上的注意力计算时间会不一致，从而在头并行部署下产生跨 GPU 的资源气泡。为了进一步减少此类气泡，我们提出了一种名为“稀疏感知头并行负载均衡”（S-HPLB）的新型注意力部署策略。在长上下文基准测试上的实验表明，S-HPLB 可以在不降低质量的情况下，使平均注意力计算延迟实现 2.88 倍的性能提升。

## Abstract
With the increasing volumes of Large Language Models (LLMs) and the expanding context lengths, attention computation has become a key performance bottleneck in LLM serving. For fast attention computation, recent practices often parallelize the attention heads on multiple GPUs, and also widely adopt attention sparsification to reduce the computation amount -- which selectively computes a subset of attention pairs under a preset sparsity budget. In this paper, we notice that attention heads of an LLM model often exhibit heterogeneous-yet-stable sparsity elasticities, which motivates us to enforce head-adaptive sparsity budgets to attain better efficiency while preserving high inference quality. Yet, from the system aspect, with heterogeneous sparsity levels, attention computation time on different heads would be inconsistent, yielding cross-GPU resource bubbles under head-parallel deployment. To further minimize such bubbles, we propose a novel attention deployment strategy called Sparsity-aware Head-Parallel Load Balance (S-HPLB). Experiments on long-context benchmark show that, S-HPLB can achieve a $2.88\times$ improvement in average attention computation latency without quality degradation.

### 1. 论文的核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）支持的上下文长度不断增加（如 128K+），**注意力机制（Attention）的计算开销**已成为推理服务的核心瓶颈，尤其是在预填充（Prefill）阶段。
*   **现有挑战**：
    *   **稀疏异构性**：不同注意力头对信息的关注度不同，有些头很稀疏（只需少量 Token 即可恢复大部分权重），有些则较稠密。现有的 Top-k 方法对所有头采用统一预算，导致资源浪费或精度损失。
    *   **系统气泡**：虽然 Top-p 等自适应方法能处理异构性，但会导致不同 GPU 上的计算负载不一致。在头并行（Head Parallelism）部署下，计算快的 GPU 必须等待计算慢的 GPU（木桶效应），产生大量资源空泡。
*   **研究目标**：设计一种算法与系统协同优化的机制，既能利用头的稀疏异构性提升精度，又能通过负载均衡消除多 GPU 并行时的延迟瓶颈。

### 2. 论文提出的方法论：S-HPLB
S-HPLB 包含两个核心组件，旨在平衡推理精度与计算效率：

*   **自适应头预算分配（Adaptive Head Budget Allocation）**：
    *   **离线特征分析**：研究发现，尽管不同输入的绝对稀疏度有别，但**各注意力头的相对稀疏性模式在不同任务和长度下是高度稳定的**。
    *   **最大-最小预算转移策略（Max-Min Strategy）**：通过离线 Profile 获取各头的稀疏特性，在总计算预算不变的前提下，迭代地将“稀疏头”（容易恢复权重的头）的多余预算转移给“稠密头”（需要更多 Token 的头），从而在不增加总计算量的情况下逼近全注意力的精度。
*   **头并行负载均衡（Head Parallel Load Balance）**：
    *   **问题建模**：将注意力头到 GPU 的分配建模为“多路划分问题（Multiway Partitioning Problem）”。
    *   **贪心启发式算法**：由于该问题是 NP-hard，S-HPLB 采用贪心算法：先将所有头按计算预算（负载）降序排列，然后依次将每个头分配给当前总负载最小的 GPU。这确保了各 GPU 间的计算耗时尽可能对齐，减少同步等待时间。

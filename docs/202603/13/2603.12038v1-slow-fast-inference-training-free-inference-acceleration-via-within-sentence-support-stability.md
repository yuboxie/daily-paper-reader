## 摘要
长上下文自回归解码仍然开销巨大，因为每个解码步骤都必须重复处理不断增长的历史记录。我们在解码过程中观察到一个一致的模式：在一个句子内，更广泛地，在一个简短的语义连贯跨度内，主要的注意力支持（attention support）通常保持基本稳定。受此观察启发，我们提出了慢-快推理（Slow-Fast Inference, SFI），这是一种无需训练的解码框架，它将生成过程解耦为频繁的低成本快步（fast steps）和偶尔的密集注意力慢步（slow steps）。快步重用紧凑的稀疏内存以实现高效解码。慢步在语义边界附近触发。在慢步中，模型重新审视更广泛的上下文，并使用选择器（Selector）刷新选定的内存，供后续快步使用。在评估的上下文长度范围内，SFI 提供了约 1.6 倍至 14.4 倍的更高解码吞吐量，同时在长上下文和长思维链（long-CoT）设置下，其质量通常与全 KV 基线持平。由于 SFI 无需训练且可直接应用于现有检查点，它为降低当代自回归推理模型在长上下文、长跨度和智能体工作负载中的推理成本提供了一条切实可行的路径。

## Abstract
Long-context autoregressive decoding remains expensive because each decoding step must repeatedly process a growing history. We observe a consistent pattern during decoding: within a sentence, and more generally within a short semantically coherent span, the dominant attention support often remains largely stable. Motivated by this observation, we propose Slow-Fast Inference (SFI), a training-free decoding framework that decouples generation into frequent low-cost fast steps and occasional dense-attention slow steps. Fast steps reuse a compact sparse memory for efficient decoding. Slow steps are triggered near semantic boundaries. At slow steps, the model revisits the broader context and uses the Selector to refresh the selected memory for subsequent fast steps. Across the evaluated context lengths, SFI delivers approximately $1.6\times$--$14.4\times$ higher decoding throughput while generally maintaining quality on par with the full-KV baseline across long-context and long-CoT settings. Because SFI is training-free and applies directly to existing checkpoints, it offers a practical path to reducing inference cost for contemporary autoregressive reasoning models in long-context, long-horizon, and agentic workloads.

### 1. 核心问题与整体含义（研究动机和背景）
在长上下文自回归解码中，随着生成长度增加，每一步解码都需要对不断增长的 KV Cache 进行全量注意力计算，导致计算开销和内存带宽压力剧增。
*   **核心观察**：作者发现 LLM 在解码过程中存在“句内支持稳定性”（Within-Sentence Support Stability）。即在一个句子或语义连贯的短跨度内，模型关注的历史令牌（Attention Support）高度重合，只有在语义边界（如句号、换行）附近，注意力焦点才会发生显著迁移。
*   **研究目标**：利用这种时间上的稳定性，打破“每一步都进行全量扫描”的传统模式，提出一种无需训练的加速框架。

### 2. 方法论：Slow-Fast Inference (SFI)
SFI 将解码过程解耦为两种步长：
*   **快步 (Fast Steps)**：这是大多数解码步骤采用的路径。模型仅对一个精简的稀疏内存进行注意力计算，该内存由三部分组成：**固定锚点 (Sink tokens)**、**局部窗口 (Recent tokens)** 和 **选定长期记忆 (Selected memory)**。在快步中，Selected 部分保持不变，从而大幅降低计算量。
*   **慢步 (Slow Steps)**：当生成触发令牌（如标点符号）或达到预设步数上限（$T_{max}$）时触发。此时模型执行全量注意力，并调用 **选择器 (Selector)** 刷新 Selected 内存。
*   **关键技术细节 (Selector)**：
    *   **KL 融合目标**：Selector 将慢步产生的注意力分布（证据）与基于键范数（Key-norm）和位置偏置的结构化先验进行融合。通过最小化逆向 KL 散度，推导出闭式解，得到连续的重要性得分。
    *   **得分精炼**：引入 **Soft-NMS**（抑制同一头内邻近位置的冗余选择）和 **跨头排他性 (Cross-head exclusivity)**（减少不同头之间重复选择同一令牌），提高稀疏内存的覆盖度和多样性。
*   **系统优化**：
    *   **异步流水线**：利用 CUDA 流将慢步的维护操作（选择与缓存重组）与下一层的计算重叠，隐藏延迟。
    *   **内存连续化算子**：将选定的 KV 令牌重组为连续缓冲区，确保 GPU 在快步解码时能进行高效的合并读取（Coalesced reads）。

### 6. 主要结论与发现
*   **吞吐量显著提升**：随着上下文长度增加，加速效果越明显。在 128K 上下文下，SFI 相比 Full-KV 实现了 **1.6x 至 14.4x** 的解码吞吐量提升。

### 7. 优点
*   **免训练且即插即用**：不需要对模型进行微调，适配现有所有自回归 LLM。
*   **事件驱动的调度**：相比于每步都更新缓存的动态方法，SFI 的慢-快调度更符合语言生成的语义结构，减少了维护频率。
*   **系统级深度优化**：不仅有算法创新，还通过异步流水线和自定义算子解决了稀疏访问导致的带宽坍塌问题。

### 8. 不足与局限
*   **触发策略依赖 Tokenizer**：慢步的触发依赖于特定的标点符号，对于不同语言或特殊格式的文本（如代码或纯数学公式），预设的触发集可能需要调整。
*   **硬件依赖性**：虽然在 B200 上表现优异，但其自定义 Triton 算子的加速效果可能受限于不同架构 GPU 的内存带宽和计算核心比例。
*   **首字延迟 (TTFT)**：SFI 主要优化解码阶段（TPOT），对于预填充（Prefill）阶段的优化相对有限（虽然提到了窗口化证据收集，但核心增益在解码）。

## 摘要
本项目复现并扩展了 Zhang 等人（2026）近期提出的“递归语言模型”（RLMs）框架。该框架通过将提示词卸载到外部 REPL 环境中，使大语言模型（LLMs）能够处理近乎无限的上下文。原论文依赖于默认的递归深度 1，并将更深层次的递归作为未来的研究方向，而本研究专门探讨了扩展递归深度的影响。利用最先进的开源智能体模型（DeepSeek v3.2 和 Kimi K2），本研究在 S-NIAH 和 OOLONG 基准测试上评估了纯 LLM、RLM（深度=1）和 RLM（深度=2）的表现。研究结果揭示了一个引人注目的现象：更深的递归会导致模型“过度思考”。虽然深度为 1 的 RLM 能有效提高复杂推理任务的准确率，但应用更深的递归（深度=2）或在简单检索任务中使用 RLM 反而会降低性能，并导致执行时间（例如从 3.6 秒增加到 344.5 秒）和 Token 成本呈指数级增长。代码和数据可在以下网址获取：https://github.com/drbillwang/rlm-reproduction

## Abstract
This project reproduces and extends the recently proposed ``Recursive Language Models'' (RLMs) framework by Zhang et al. (2026). This framework enables Large Language Models (LLMs) to process near-infinite contexts by offloading the prompt into an external REPL environment. While the original paper relies on a default recursion depth of 1 and suggests deeper recursion as a future direction, this study specifically investigates the impact of scaling the recursion depth. Using state-of-the-art open-source agentic models (DeepSeek v3.2 and Kimi K2), I evaluated pure LLM, RLM (depth=1), and RLM (depth=2) on the S-NIAH and OOLONG benchmarks. The findings reveal a compelling phenomenon: Deeper recursion causes models to ``overthink''. While depth-1 RLMs effectively boost accuracy on complex reasoning tasks, applying deeper recursion (depth=2) or using RLMs on simple retrieval tasks paradoxically degrades performance and exponentially inflates execution time (e.g., from 3.6s to 344.5s) and token costs. Code and data are available at: https://github.com/drbillwang/rlm-reproduction

#### 1. 核心问题与研究动机
*   **核心问题**：验证并扩展“递归语言模型”（RLM）框架在处理超长上下文时的有效性，特别是探索增加递归深度（Recursion Depth）对模型性能、推理成本和执行效率的影响。
*   **研究背景**：传统的长上下文处理受限于显存和计算复杂度。RLM 提出将长提示词作为外部 REPL（交互式解释器）环境中的持久变量，允许模型通过编程方式自我调用和分解任务。原研究主要关注深度为 1 的情况，本论文旨在探讨“更深是否更好”。

#### 2. 方法论：核心思想与技术细节
*   **核心思想**：将 LLM 嵌入到一个 Read-Eval-Print Loop (REPL) 环境中。模型不再一次性读取全部上下文，而是通过生成代码来检索、切片和递归处理输入数据。
*   **关键技术**：
    *   **任务分解**：根模型（Root LM）将复杂查询分解为子任务。
    *   **递归调用**：子任务可以再次触发 RLM 流程（深度 > 1），或者作为标准 LLM 调用（深度 = 1）。
    *   **环境隔离**：长文本存储在外部环境中，模型仅通过 API 或编程指令与其交互，从而绕过原生上下文窗口限制。
    *   **解析优化**：针对 Kimi K2 等模型增加了对 `<thinking>` 标签的过滤处理，以确保 REPL 能够正确解析最终答案。

#### 6. 主要结论与发现
*   **“思考”有益**：对于复杂推理任务（OOLONG），深度为 1 的 RLM 显著提升了模型表现（如 DeepSeek 从 0% 提升至 42.1%）。
*   **“过度思考”有害**：
    *   在简单任务（S-NIAH）中，RLM 反而导致准确率下降（100% $\rightarrow$ 70%）。

    2.  **格式崩溃**：模型混淆了 REPL 内部代码和最终输出格式。
    3.  **表演式推理**：模型陷入无休止的自我验证循环，产生大量冗余步骤。

## 摘要
随着大语言模型（LLMs）越来越多地用于长时任务，维持有效的长期记忆已成为一个关键挑战。当前的方法通常面临成本与准确性之间的权衡。简单的存储方法往往无法检索到相关信息，而复杂的索引方法（如记忆图）需要繁重的计算并可能导致信息丢失。此外，依靠工作 LLM 处理所有记忆在计算上是昂贵且缓慢的。为了解决这些局限性，我们提出了 MemSifter，这是一个将记忆检索过程卸载到小规模代理模型的新型框架。MemSifter 不是增加主工作 LLM 的负担，而是使用一个较小的模型在检索必要信息之前对任务进行推理。这种方法在索引阶段不需要繁重的计算，并且在推理过程中增加的开销极小。为了优化代理模型，我们引入了一种专门针对记忆的强化学习（RL）训练范式。我们基于工作 LLM 完成任务的实际表现，设计了一种面向任务结果的奖励机制。该奖励通过与工作 LLM 的多次交互来衡量检索到的记忆的实际贡献，并通过阶梯式递减的贡献度来区分检索排名。此外，我们采用了课程学习（Curriculum Learning）和模型合并（Model Merging）等训练技术来提高性能。我们在包括深度研究（Deep Research）任务在内的八个 LLM 记忆基准测试上评估了 MemSifter。结果表明，我们的方法在检索准确性和最终任务完成度方面均达到或超过了现有最先进方法的性能。MemSifter 为长期 LLM 记忆提供了一种高效且可扩展的解决方案。我们已经开源了模型权重、代码和训练数据，以支持进一步的研究。

## Abstract
As Large Language Models (LLMs) are increasingly used for long-duration tasks, maintaining effective long-term memory has become a critical challenge. Current methods often face a trade-off between cost and accuracy. Simple storage methods often fail to retrieve relevant information, while complex indexing methods (such as memory graphs) require heavy computation and can cause information loss. Furthermore, relying on the working LLM to process all memories is computationally expensive and slow. To address these limitations, we propose MemSifter, a novel framework that offloads the memory retrieval process to a small-scale proxy model. Instead of increasing the burden on the primary working LLM, MemSifter uses a smaller model to reason about the task before retrieving the necessary information. This approach requires no heavy computation during the indexing phase and adds minimal overhead during inference. To optimize the proxy model, we introduce a memory-specific Reinforcement Learning (RL) training paradigm. We design a task-outcome-oriented reward based on the working LLM's actual performance in completing the task. The reward measures the actual contribution of retrieved memories by mutiple interactions with the working LLM, and discriminates retrieved rankings by stepped decreasing contributions. Additionally, we employ training techniques such as Curriculum Learning and Model Merging to improve performance. We evaluated MemSifter on eight LLM memory benchmarks, including Deep Research tasks. The results demonstrate that our method meets or exceeds the performance of existing state-of-the-art approaches in both retrieval accuracy and final task completion. MemSifter offers an efficient and scalable solution for long-term LLM memory. We have open-sourced the model weights, code, and training data to support further research.

随着大语言模型（LLM）被用于处理长达数天甚至数月的长程任务，如何高效管理和检索长期记忆成为核心瓶颈。
*   **现有痛点：** 
    *   **简单检索（Vanilla）：** 依赖向量相似度，准确率低且无法处理复杂逻辑。
    *   **结构化增强（如记忆图）：** 索引阶段计算量巨大，且摘要过程会导致细节丢失。
    *   **长上下文直接处理：** 让昂贵的主模型（Working LLM）直接阅读所有历史记录，计算成本极高且速度慢。
*   **研究目标：** 提出一种名为 **MemSifter** 的框架，将记忆检索的“推理压力”从主模型卸载（Offload）到一个轻量级的代理模型（Proxy Model）上，实现高精度与低成本的平衡。

### 2. 方法论
MemSifter 的核心思想是**“先推理，再检索”**，并利用任务结果直接指导检索优化。

*   **核心流程：**
    1.  **记忆分段：** 将历史记录按主题切分为多个 Session，并赋予唯一 ID。
    2.  **代理推理（Think-and-Rank）：** 代理模型（如 4B 参数模型）接收当前任务和历史 Session，先生成推理链（`<think>` 标签），分析任务依赖哪些历史，然后输出 Top-k 的 Session ID 排名。
    3.  **主模型执行：** 主模型仅接收代理模型筛选出的精简记忆片段，完成最终任务。

*   **关键技术：任务结果驱动的强化学习（RL）**
    *   **边际效用奖励（Marginal Utility Reward）：** 奖励不再基于语义相似度，而是基于主模型的表现提升。公式逻辑为：`奖励 = (有记忆时的任务得分) - (无记忆时的基准得分)`。这确保了检索器只为真正解决知识缺口的记忆买单。
    *   **排名敏感奖励（Rank-Sensitive Reward）：** 借鉴 DCG（贴现增益）思想，通过阶梯式递减的权重（$w_n$），激励代理模型将最关键的记忆排在最前面，以适应 LLM 的注意力偏好。
    *   **优化协议：** 采用迭代训练，结合**课程学习**（动态选择难度适中的样本）和**模型合并**（Model Merging，平滑 RL 训练波动）。

### 6. 主要结论与发现
*   **性能卓越：** MemSifter 在所有 8 个数据集上均达到或超过了现有最先进方法的水平。
*   **语义相关性 ≠ 任务效用：** 实验证明，传统的基于相似度的检索往往会引入逻辑无关的噪声，而 MemSifter 筛选出的记忆更具“推理价值”。
*   **小模型胜大模型：** 经过专门 RL 优化的 4B 代理模型，在检索任务上的表现优于直接使用 600B+ 参数的长文本模型处理全量数据。
*   **效率提升：** 相比于直接处理 128k 窗口，MemSifter 将主模型的处理量压缩至 2k 左右，大幅降低了推理延迟。

### 7. 优点
*   **架构解耦：** 巧妙地将“记忆筛选”与“任务生成”分离，保护了主模型的计算资源。
*   **标签无关：** 采用结果驱动的 RL，解决了长期记忆任务中“难以获取精准排名标注”的难题。
*   **推理增强：** 代理模型自带的“思考”过程显著提升了对隐性依赖和多跳逻辑的捕捉能力。

### 8. 不足与局限
*   **冷启动依赖：** 虽然强调无标注，但在 RL 初期仍需少量监督数据进行“热身”，否则模型难以收敛。
*   **多模态缺失：** 目前主要针对文本交互历史，尚未扩展到包含图像、视频等多模态记忆的检索。
*   **代理模型瓶颈：** 系统的上限受限于代理模型的推理能力，若任务逻辑极其复杂，4B 规模的模型可能仍存在理解偏差。

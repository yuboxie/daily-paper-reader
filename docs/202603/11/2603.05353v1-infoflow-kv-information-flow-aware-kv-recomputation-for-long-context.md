## 摘要
用于长上下文问答的检索增强生成 (RAG) 受限于在大规模检索上下文上的推理时预填充 (prefilling)。一种常见的策略是为单个文档预计算键值 (KV) 缓存，并选择性地重计算一小部分 token 以恢复全局因果依赖，但现有方法依赖于启发式规则或表示差异，而没有建模所选 token 是否能有效影响生成。我们将选择性 KV 重计算视为一个信息流问题，并证明在推理一致的 RoPE 几何结构下计算时，来自查询 (query) 的简单注意力范数 (attention-norm) 信号可以可靠地识别出既具有语义相关性、又在结构上处于信息传播位置的 token。因此，我们为检索到的分块重建了全局位置分配，并引入了一种信息流引导的分块重排序策略。在 LLM 和 VLM 基准测试上的实验表明，在相当的效率预算下，该方法相比先前方法取得了一致的性能提升。

## Abstract
Retrieval-augmented generation (RAG) for long-context question answering is bottlenecked by inference-time prefilling over large retrieved contexts. A common strategy is to precompute key-value (KV) caches for individual documents and selectively recompute a small subset of tokens to restore global causal dependencies, but existing methods rely on heuristics or representation discrepancies without modeling whether selected tokens can effectively influence generation. We cast selective KV recomputation as an information flow problem and show that a simple attention-norm signal from the query reliably identifies tokens that are both semantically relevant and structurally positioned to propagate information, when computed under an inference-consistent RoPE geometry. We therefore reconstruct global positional assignments for retrieved chunks and introduce an information-flow-guided chunk reordering strategy. Experiments on LLM and VLM benchmarks demonstrate consistent gains over prior methods under comparable efficiency budgets.

### 1. 核心问题与研究动机
*   **核心问题**：在长上下文检索增强生成（RAG）场景下，推理过程中的“预填充（Prefilling）”阶段是主要的性能瓶颈。
*   **研究背景**：为了提高效率，通常会离线预计算单个文档的 KV 缓存。然而，当这些分块（Chunks）在推理时组合在一起时，由于每个分块是独立计算的（局部因果掩码和局部位置编码），丢失了跨文档的全局因果依赖。
*   **动机**：现有的选择性重计算方法（如 CacheBlend 或 EPIC）多依赖于浅层表示差异或固定的位置启发式规则，未能准确建模哪些 token 真正能有效影响最终的生成结果。本文旨在从“信息流”的角度，识别并重计算那些对查询（Query）最具影响力的关键 token。

### 2. 方法论
*   **核心思想**：将选择性 KV 重计算建模为一个信息流问题，利用查询到上下文 token 的**注意力范数（Attention-Norm）**作为识别关键 token 的信号。
*   **关键技术细节**：
    *   **注意力范数准则**：计算从 Prompt 到上下文 token 的注意力权重矩阵 $A$，定义每个 token 的重要性得分 $s_j = \sum_{i=1}^{M} A_{ij}$（$M$ 为 Prompt 长度）。选择得分最高的 Top-k 个 token 进行重计算。
    *   **推理一致的 RoPE 几何（Global RoPE）**：研究发现，只有在全局位置重建（Global Positional Reconstruction）的 RoPE 布局下计算注意力范数，才能准确识别关键 token。
    *   **全局位置重建**：在重计算前，将所有检索到的分块按全局顺序排列，并重新分配全局位置索引，以消除预填充时的局部位置偏差。
    *   **信息流引导的分块重排序**：根据识别出的关键 token 分布，将包含更多重要信息的文档块移动到靠近 Prompt 的位置，以进一步增强注意力效果。
    *   **算法流程**：1. 独立预填充分块；2. 拼接分块与 Prompt；3. 在全局 RoPE 下计算注意力范数并筛选 Top-k token；4. 对选定 token 进行全局因果掩码下的 KV 重计算；5. 执行自回归解码。

### 7. 优点

*   **多模态支持**：是少数能同时在 LLM 和 VLM 上证明有效的 KV 重计算方案。

### 8. 不足与局限
*   **硬件利用率挑战**：选择性重计算会产生不规则的注意力掩码（既非全稠密也非严格因果），现有的优化内核（如 FlashAttention）对此类稀疏模式的支持尚不完美，导致实际硬件加速比低于理论计算量的缩减。

## 摘要
提示词长度是智能体大语言模型（LLM）工作负载中的主要瓶颈，其中重复的推理步骤和多轮调用循环会产生巨大的预填充成本。最近关于投机性预填充的研究表明，基于注意力的 Token 重要性估计可以实现免训练的提示词压缩，但这假设存在一个与目标模型共享相同分词器（tokenizer）的草稿模型。然而在实践中，智能体流水线经常使用没有同系列小型草稿模型的模型。在这项工作中，我们研究了跨家族投机性预填充，即使用来自一个模型家族的轻量级草稿模型为来自不同家族的目标模型执行提示词压缩。利用与先前工作相同的投机性预填充机制，我们评估了一系列跨家族的草稿-目标组合，包括 Qwen、LLaMA 和 DeepSeek 模型。在广泛的任务中，我们发现尽管草稿模型和目标模型在模型架构和分词器上存在差异，但基于注意力的 Token 重要性估计在不同模型家族之间仍能可靠地迁移。跨模型提示词压缩在很大程度上保留了全提示词基线性能的 90~100%，在某些情况下，由于去噪效应，准确率甚至略有提高，同时显著降低了首字延迟（TTFT）。这些结果表明，投机性预填充主要依赖于任务先验和语义结构，因此可以作为一种通用的提示词压缩原语。我们讨论了研究结果对智能体系统的启示，在这些系统中，重复的长上下文推理和异构模型栈使得跨模型提示词压缩既必要又实用。

## Abstract
Prompt length is a major bottleneck in agentic large language model (LLM) workloads, where repeated inference steps and multi-call loops incur substantial prefill cost. Recent work on speculative prefill demonstrates that attention-based token importance estimation can enable training-free prompt compression, but this assumes the existence of a draft model that shares the same tokenizer as the target model. In practice, however, agentic pipelines frequently employ models without any smaller in-family draft model. In this work, we study cross-family speculative prefill, where a lightweight draft model from one model family is used to perform prompt compression for a target model from a different family. Using the same speculative prefill mechanism as prior work, we evaluate a range of cross-family draft-target combinations, including Qwen, LLaMA, and DeepSeek models. Across a broad diversity of tasks, we find that attention-based token importance estimation transfers reliably across different model families despite differences in model architectures and tokenizers between draft and target models. Cross-model prompt compression largely retains 90~100% of full-prompt baseline performance and, in some cases, slightly improves accuracy due to denoising effects, while delivering substantial reductions in time to first token (TTFT). These results suggest that speculative prefill depends mainly on task priors and semantic structure, thus serving as a generalizable prompt compression primitive. We discuss the implications of our findings for agentic systems, where repeated long-context inference and heterogeneous model stacks make cross-model prompt compression both necessary and practical.

### 1. 核心问题与整体含义（研究动机和背景）
在智能体（Agentic）工作流中，LLM 需要处理包含推理轨迹、检索文档和执行日志的长上下文。这导致**预填充（Prefill）阶段**成为延迟和吞吐量的主要瓶颈。
*   **研究动机**：现有的“投机性预填充”技术虽然能通过小型草稿模型（Draft Model）预测 Token 重要性来压缩提示词，但其前提是草稿模型与目标模型必须属于**同一家族且共享相同的分词器（Tokenizer）**。
*   **核心问题**：在实际应用中，许多顶尖模型（如 DeepSeek-V3/R1）缺乏同家族的小型模型，且智能体系统往往采用异构模型栈。本文旨在探索：**基于注意力的 Token 重要性估计是否能在不同模型家族之间迁移？**

### 2. 论文提出的方法论
论文提出了**跨家族投机性预填充**，这是一种免训练（Training-free）的提示词压缩技术。
*   **核心思想**：利用不同家族的轻量级草稿模型来识别输入提示词中的显著（Salient）部分，并将压缩后的文本传递给目标模型。
*   **关键技术细节**：
    1.  **重要性估计**：草稿模型对全量提示词进行前向传播，提取注意力张量。采用 **Max-Mean 聚合策略**（在层和头维度取最大值，在 Lookahead 维度取平均值）计算每个 Token 的重要性得分。
    2.  **分块选择**：为降低方差，将 Token 序列划分为固定大小的块（Chunks），计算块平均分并保留 Top-K 块。
    3.  **跨家族适配**：
        *   **文本级中转**：在草稿域选定块后，将其映射回原始文本，并插入特定分隔符（如 `[...]` 或 `// omitted`）。
        *   **重新分词与位置编码**：目标模型对压缩后的文本重新分词。关键改进是**分配新的连续位置 ID**，而非恢复原始索引，从而避开了异构分词器之间的位置对齐难题。
*   **算法流程**：输入长提示词 -> 草稿模型计算注意力 -> 聚合得分 -> 分块并选择 Top-K -> 拼接文本 -> 目标模型分词 -> 目标模型推理。

### 6. 论文的主要结论与发现
1.  **跨家族迁移性**：注意力驱动的 Token 重要性信号在不同模型家族间具有高度可迁移性，不依赖特定的分词器或架构。
2.  **性能保持**：在大多数任务中，压缩后的提示词能保留 **90-100%** 的基线准确率。
3.  **去噪效应**：在 RULER 等检索任务中，压缩提示词的效果有时甚至**优于全量提示词**，因为压缩过程移除了无关的噪声上下文。
4.  **效率提升**：在 128k 上下文下，实现了高达 **18 倍** 的 TTFT 缩减。
5.  **突破部署限制**：允许在硬件内存受限（如仅支持 32k 目标上下文）的情况下，通过长上下文草稿模型处理 128k 输入。

### 7. 优点
*   **通用性极强**：打破了“同家族模型”的限制，使任何轻量级模型都能为闭源或大型模型提供压缩服务。
*   **完全免训练**：无需微调，即插即用，极大地降低了部署门槛。
*   **硬件友好**：通过减少预填充阶段的计算量，显著降低了对显存和算力的需求。
*   **逻辑简化**：通过重新分配连续位置 ID，巧妙解决了跨分词器对齐的复杂问题。

### 8. 不足与局限
*   **代码任务敏感性**：在代码调试（Code Debug）任务中，当压缩率过高（如仅保留 15%）时，性能下降较快（约下降 12-16%），说明代码逻辑对细粒度语法结构的依赖性更高。
*   **结构依赖性**：目前的压缩策略是通用的，缺乏对特定任务（如代码依赖关系）的结构感知。
*   **草稿模型开销**：虽然草稿模型很小，但对于极短的提示词，草稿模型的额外前向传播可能会抵消压缩带来的收益（主要适用于长文本）。

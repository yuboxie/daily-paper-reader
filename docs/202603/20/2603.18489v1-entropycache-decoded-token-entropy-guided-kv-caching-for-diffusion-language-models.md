## 摘要
基于扩散的大语言模型 (dLLMs) 依赖于双向注意力机制，这阻碍了无损 KV 缓存的实现，并要求在每个去噪步骤中进行完整的正向传递。现有的近似 KV 缓存方法通过选择性地更新缓存状态来降低这一成本，但其决策开销随上下文长度或模型深度而增加。我们提出了 EntropyCache，这是一种无需训练的 KV 缓存方法，它利用新解码 Token 分布的最大熵作为恒定成本信号，用于决定何时重新计算。我们的设计基于两个经验观察：(1) 解码 Token 的熵与 KV 缓存漂移相关，为缓存陈旧度提供了一个廉价的代理指标；(2) 解码 Token 的特征波动在取消掩码后会持续多个步骤，这促使我们重新计算最近解码的 $k$ 个 Token。跳过或重新计算的决策每步仅需 $O(V)$ 的计算量，与上下文长度和模型规模无关。在 LLaDA-8B-Instruct 和 Dream-7B-Instruct 上的实验表明，EntropyCache 在标准基准测试中实现了 15.2x-26.4x 的加速，在思维链基准测试中实现了 22.4x-24.1x 的加速，同时具有竞争力的准确率，且决策开销仅占推理时间的 0.5%。代码可在 https://github.com/mscheong01/EntropyCache 获取。

## Abstract
Diffusion-based large language models (dLLMs) rely on bidirectional attention, which prevents lossless KV caching and requires a full forward pass at every denoising step. Existing approximate KV caching methods reduce this cost by selectively updating cached states, but their decision overhead scales with context length or model depth. We propose EntropyCache, a training-free KV caching method that uses the maximum entropy of newly decoded token distributions as a constant-cost signal for deciding when to recompute. Our design is grounded in two empirical observations: (1) decoded token entropy correlates with KV cache drift, providing a cheap proxy for cache staleness, and (2) feature volatility of decoded tokens persists for multiple steps after unmasking, motivating recomputation of the $k$ most recently decoded tokens. The skip-or-recompute decision requires only $O(V)$ computation per step, independent of context length and model scale. Experiments on LLaDA-8B-Instruct and Dream-7B-Instruct show that EntropyCache achieves $15.2\times$-$26.4\times$ speedup on standard benchmarks and $22.4\times$-$24.1\times$ on chain-of-thought benchmarks, with competitive accuracy and decision overhead accounting for only $0.5\%$ of inference time. Code is available at https://github.com/mscheong01/EntropyCache.

#### 1. 核心问题与研究动机
*   **核心问题**：扩散语言模型（dLLMs）采用**双向（非因果）注意力机制**，这导致其无法像自回归模型（AR）那样实现无损的 KV 缓存。在推理的每个去噪步骤中，即使只改变一个 Token，也会影响所有位置的表示，迫使模型进行全量的正向传播，计算开销随序列长度线性增长。
*   **研究动机**：现有的近似 KV 缓存方法（如 d2Cache 或 Elastic-Cache）虽然能减少计算量，但它们的“决策开销”（即决定哪些部分需要重新计算的计算量）往往随着上下文长度、模型深度或隐藏层维度增加，限制了实际的加速效果。本文旨在寻找一种**计算成本极低且与序列长度无关**的信号，来动态指导 KV 缓存的更新。

#### 2. 方法论
EntropyCache 是一种**无需训练（Training-free）**的动态 KV 缓存方法，其核心思想基于两个关键的经验观察：
*   **核心思想**：
    1.  **熵作为漂移代理**：新解码 Token 分布的**最大熵（Maximum Entropy）**与 KV 缓存的漂移程度（Staleness）高度正相关。高熵意味着模型对当前预测不确定，会引入更多新信息，导致缓存失效；低熵则意味着预测稳定，缓存可复用。
    2.  **特征波动持久性**：Token 在取消掩码（Unmasking）后的特征波动会持续多个步骤，而非仅在解码瞬间。
*   **关键技术细节与流程**：
    1.  **熵引导的跳过决策**：在每一步去噪后，计算新解码 Token 分布的最大熵 $E_{t+1}$。若 $E_{t+1} > \tau$（阈值），则在下一步触发**全量重新计算**；否则，执行**部分重新计算**。
    2.  **最近 Token 重新计算策略**：在部分重新计算模式下，除了计算当前的 Mask Token 外，还固定重新计算**最近解码的 $k$ 个 Token**。这能有效捕捉 Token 解码后的多步不稳定性。
    3.  **算法复杂度**：决策过程仅涉及 $O(V)$（$V$ 为词表大小）的标量计算，完全独立于上下文长度 $L$ 和模型深度，决策开销几乎可以忽略不计。

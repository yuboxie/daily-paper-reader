## 摘要
Engram 条件记忆通过将静态知识查找与动态计算解耦，已成为大语言模型（LLM）中一个极具前景的组件。由于 Engram 表现出稀疏的访问模式并支持预取，其庞大的嵌入表非常适合卸载到低层级内存中。在本文中，我们提出使用 Compute Express Link (CXL) 内存池进行 Engram 存储。与 RDMA 相比，CXL 提供了 Engram 极小且离散的检索模式所需的细粒度、低延迟访问。我们将基于 CXL 的 Engram 池集成到 SGLang 中，实现了接近 DRAM 的端到端性能。这为未来集成 Engram 的大语言模型提供了一种可扩展且具有成本效益的存储解决方案，且不会损害推理性能。

## Abstract
Engram conditional memory has emerged as a promising component for LLMs by decoupling static knowledge lookup from dynamic computation. Since Engram exhibits sparse access patterns and supports prefetching, its massive embedding tables are well-suited for offloading to lower-tier memory. In this paper, we propose using Compute Express Link (CXL) memory pool for Engram storage. Compared to RDMA, CXL provides fine-grained and low-latency access required by minimal and discrete retrieval patterns of Engram. We integrate the CXL-based Engram pool into SGLang, achieving near-DRAM end-to-end performance. This provides a scalable and cost-efficient storage solution for future Engram-integrated LLMs without compromising inference performance.

### 1. 核心问题与研究动机
*   **核心问题**：Engram 架构（由 DeepSeek 提出）通过引入大规模 N-Gram 嵌入表（条件记忆）来解耦静态知识查找与动态计算，显著提升了模型性能。然而，这些嵌入表体积巨大（可达数百 GB），超出了 GPU 显存（HBM）的承载能力。
*   **研究动机**：传统的 RDMA 远程内存池在处理 Engram 这种**极小规模（每 token 仅约 5KB）、高度离散且稀疏**的访问模式时，协议开销大、延迟高。论文旨在寻找一种既能扩展内存容量，又能保持低延迟检索的存储方案，以支持下一代集成 Engram 的 LLM 推理。

### 2. 方法论
*   **核心思想**：利用 CXL 2.0/3.0 协议的硬件级 load/store 原语，构建一个跨节点的共享内存池，将 Engram 嵌入表从昂贵的 GPU 显存卸载到高性价比的 CXL 内存中。
*   **关键技术细节**：
    *   **CXL 内存池架构**：通过 CXL Switch 连接多个计算节点和一个中心化的 CXL 内存池，实现多主机共享访问。
    *   **优化的访问例程**：
        *   **CXL $\rightarrow$ CPU**：使用 OpenMP 多线程并行执行 `memcpy`，充分利用 CXL 的带宽。
        *   **CXL $\rightarrow$ GPU**：开发了自定义 CUDA 内核，通过 `cudaHostRegister` 将 CXL 内存映射到 GPU 地址空间，实现点对点（P2P）DMA 传输，绕过 CPU 频繁干预。
    *   **异步预取机制**：利用 Engram 模块仅存在于特定层（如第 2 层和第 15 层）且不依赖中间隐藏状态的特性，在推理开始时即启动异步检索，将内存访问延迟与前序 Transformer 层的计算重叠（Overlap）。

### 6. 主要结论与发现
*   **延迟优势**：CXL 的检索延迟远低于 RDMA，且非常接近本地 DRAM。在 Batch Size 为 256 时，CXL 检索延迟在 100-200 $\mu s$ 级别，满足 Engram 的预取时间窗口要求。
*   **性能表现**：集成 CXL 内存池后，SGLang 的端到端推理吞吐量达到了**接近原生 DRAM（Near-DRAM）**的水平，性能损失极小。
*   **成本效益**：随着模型规模（如 400B Engram）和节点数量增加，CXL 共享内存池方案比为每个节点配置全量 DRAM 具有显著的成本优势（可节省数万美元）。

### 7. 优点
*   **针对性强**：精准捕捉到了 Engram 稀疏、离散访问与 CXL 细粒度访问特性的高度契合。
*   **系统集成度高**：不仅有理论分析，还通过自定义 CUDA Kernel 和 SGLang 集成实现了完整的原型系统。
*   **前瞻性**：为未来超大规模知识增强型 LLM 的硬件架构提供了切实可行的演进方向。

### 8. 不足与局限
*   **模型权重缺失**：由于 DeepSeek Engram 官方权重未开源，实验使用了 Qwen3 模型进行性能模拟，虽然能反映系统开销，但缺乏真实业务场景下的精度验证。
*   **硬件普及度**：CXL 2.0/3.0 硬件目前仍处于早期部署阶段，大规模集群的稳定性及多租户下的资源竞争（如 KV Cache 与 Engram 共存）仍需进一步研究。

## 摘要
语言模型难以泛化到预训练上下文长度之外，这限制了长程推理和检索能力。在长上下文数据上进行持续预训练虽有帮助，但由于注意力机制（Attention）的二次方缩放特性，其成本非常高昂。我们观察到，大多数标记（token）并不需要对整个序列进行（全局）注意力计算，而可以依赖局部上下文。基于此，我们提出了 L2A（Learning To Attend），这是一种通过决定何时调用全局注意力来实现条件化（逐标记）长程记忆访问的层。我们在 Qwen 2.5 和 Qwen 3 模型上评估了 L2A，将其有效上下文长度从 32K 扩展到了 128K 标记。L2A 在跳过约 80% 标记的全局注意力计算的同时，性能与标准长上下文训练相比差距在 3% 以内，优于之前的基准方法。我们还设计了定制的 Triton 内核，以在 GPU 上高效实现这种逐标记的条件注意力，与 FlashAttention 相比，训练吞吐量和首字生成时间（time-to-first-token）提升了约 2 倍。此外，L2A 支持对高度稀疏的全局注意力层进行训练后剪枝，在性能损失微乎其微的情况下，将 KV 缓存内存减少了高达 50%。

## Abstract
Language models struggle to generalize beyond pretraining context lengths, limiting long-horizon reasoning and retrieval. Continued pretraining on long-context data can help but is expensive due to the quadratic scaling of Attention. We observe that most tokens do not require (Global) Attention over the entire sequence and can rely on local context. Based on this, we propose L2A (Learning To Attend), a layer that enables conditional (token-wise) long-range memory access by deciding when to invoke global attention. We evaluate L2A on Qwen 2.5 and Qwen 3 models, extending their effective context length from 32K to 128K tokens. L2A matches the performance of standard long-context training to within 3% while skipping Global Attention for $\sim$80% of tokens, outperforming prior baselines. We also design custom Triton kernels to efficiently implement this token-wise conditional Attention on GPUs, achieving up to $\sim$2x improvements in training throughput and time-to-first-token over FlashAttention. Moreover, L2A enables post-training pruning of highly sparse Global Attention layers, reducing KV cache memory by up to 50% with negligible performance loss.

### 论文总结：Learning When to Attend: Conditional Memory Access for Long-Context LLMs

#### 1. 核心问题与整体含义（研究动机和背景）
大语言模型（LLMs）在处理长文本时面临两个主要挑战：一是**计算成本高昂**，标准注意力机制（Attention）的计算复杂度随序列长度呈二次方增长；二是**泛化能力有限**，模型难以处理超出预训练长度的上下文。
研究人员观察到，并非所有标记（token）都需要全局注意力。例如，处理“因此季度收入是”仅需局部上下文，而“如第2节所述”则需检索数千个标记前的记忆。论文的核心动机是：**能否让模型学会逐标记地决定何时访问长程记忆，从而在不损失性能的前提下大幅降低计算开销？**

#### 2. 方法论：核心思想与技术细节
论文提出了 **L2A (Learning To Attend)** 机制，将全局注意力视为一种“按需调用”的长程记忆检索。

*   **核心架构**：L2A 层由三个模块组成：
    1.  **局部注意力（Local Attention）**：使用滑动窗口注意力（SWA）捕捉近期依赖。
    2.  **路由决策器（Router）**：一个轻量级的线性投影层，根据局部表示计算得分 $\hat{d}_t$，并通过 Sigmoid 函数生成离散决策 $d_t \in \{0, 1\}$。
    3.  **全局注意力（Global Attention）**：仅当 $d_t=1$ 时，该 token 才会对全量上下文执行昂贵的全局注意力计算。
*   **训练策略**：
    *   **直通估计器（STE）**：由于路由决策是离散不可导的，训练时使用 STE 将梯度传回路由模块。
    *   **稀疏正则化**：在损失函数中加入惩罚项，鼓励模型尽量减少全局注意力的调用。
    *   **防止路由崩溃**：在训练中以小概率（如 10%）强制所有 token 调用全局注意力，确保路由模块能接收到足够的学习信号。
*   **硬件优化**：开发了定制的 **Triton 内核**，通过重新排列活跃 token 形成连续缓冲区，实现了硬件感知的稀疏计算加速。
*   **推理优化（层剪枝）**：发现约 50% 的层几乎不调用全局注意力，通过在训练后剪掉这些层的全局模块，可减少高达 50% 的 KV 缓存内存。

*   **性能保持**：L2A 在仅使用约 **20%** 全局计算的情况下，性能达到了全量微调（CLP）的 **97% 以上**。

#### 7. 优点
*   **动态自适应**：不同于 NSA 等固定稀疏模式的方法，L2A 能根据输入内容的复杂度动态分配计算资源。
*   **端到端加速**：不仅有算法创新，还配套了高效的 Triton 内核，解决了“算法稀疏但不提速”的常见痛点。
*   **KV 缓存优化**：提出的层剪枝策略为长文本推理的内存瓶颈提供了切实可行的解决方案。

#### 8. 不足与局限
*   **KV 缓存存储**：虽然计算减少了，但在未剪枝的层中，仍需存储所有 token 的 KV 向量，因为无法预知未来哪个 token 会需要它们。

*   **模型覆盖**：实验主要集中在 Qwen 系列，尚未在 Llama 或其他架构上广泛验证。
